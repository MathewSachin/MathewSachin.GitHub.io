If your bookmarklet breaks every two weeks, it is not an automation tool — it is a coin flip.

## Target query
How do I write bookmarklets that keep working across layout changes and mobile browsers?

## Promise
You will leave with a practical checklist to make bookmarklets resilient, readable, and reusable.

## Pattern 1: Stable selectors over brittle CSS chains
Use semantic anchors (`aria-*`, text labels, URL patterns) before deep CSS selectors.

## Pattern 2: Failure fallback paths
Add guard clauses and alternate strategies instead of crashing on first miss.

## Pattern 3: Mobile compatibility by default
Test on mobile bookmark execution constraints and avoid desktop-only assumptions.

## Pattern 4: CSP and sandbox caveats
Know when bookmarklets can modify DOM only versus when network/script paths are blocked.
