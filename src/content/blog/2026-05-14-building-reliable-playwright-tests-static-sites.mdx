A passing test suite once is luck. A passing test suite in CI every day is engineering.

## Target query
How do I make Playwright tests stable for static sites with client-hydrated islands?

## Promise
You will get a repeatable pattern for reducing hydration race failures and CI-only flakes.

## Hydration timing and navigation strategy
Prefer deterministic ready signals over assumptions about load events.

## Retry patterns that do not hide bugs
Use bounded retries around fragile interactions, then assert hard outcomes.

## CI stability and observability
Capture enough context (traces, screenshots, logs) to make failed runs debuggable in minutes.

## Local debugging workflow
Reproduce CI timing behavior locally before changing assertions.

## Actionable takeaway
Write one explicit “ready” condition per page and make every interaction wait on that condition before acting.

If your UI behavior changed, validate the flow with `npm run test:e2e` before shipping.
