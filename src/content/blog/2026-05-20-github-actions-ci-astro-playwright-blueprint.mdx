Fast CI is nice. Fast CI that still catches regressions is better.

## Target query
What is the right GitHub Actions design for Astro build + Playwright validation?

## Promise
You will get a practical pipeline structure that is fast, debuggable, and maintainable.

## Build artifact strategy
Build once, test many; avoid recomputing the same site in every job.

## Pipeline split for e2e reliability
Separate deterministic build checks from browser-runtime checks.

## Debugging failed jobs quickly
Use artifacts, traces, and focused logs to cut triage time.

## Cost and runtime optimization
Cache wisely and remove redundant work without sacrificing confidence.
