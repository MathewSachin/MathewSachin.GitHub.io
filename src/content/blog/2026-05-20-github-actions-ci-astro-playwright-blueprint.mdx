*Fast CI is easy if you skip useful checks. Reliable CI is easy if you accept slow feedback. The real challenge is designing a pipeline that is both fast and trustworthy.*

## Target query

How should I structure GitHub Actions for an Astro static site with Playwright e2e tests?

## Promise

You’ll get a practical workflow blueprint that balances speed, observability, and confidence.

## Core principle: build once, reuse everywhere

The highest-leverage CI optimization is artifact reuse.

Pipeline shape:

1. install + build site once
2. publish `dist/` artifact
3. downstream jobs (tests/previews) consume same artifact

This avoids duplicate builds and removes “it passed in build job but failed in test job due to different output” drift.

## Split checks by failure domain

Separate jobs by what can break:

- **build/unit checks**: compile/content/schema/runtime-independent logic
- **browser checks**: Playwright against built artifact
- **deploy checks**: only after quality gates pass

When jobs are scoped well, failures are faster to diagnose.

## Debugging failed jobs without guesswork

Always preserve:

- test traces/screenshots/videos
- build logs
- generated artifact metadata

If an e2e test fails, the review path should be obvious: open trace, inspect state, identify failing assumption.

## Runtime and cost optimization levers

- dependency caching with lockfile keying
- artifact reuse across jobs
- conditional workflow triggers for docs/content-only changes (where appropriate)
- matrix pruning to only meaningful browser/runtime combinations

Optimization rule: remove duplicated work first, micro-tune second.

## CI hygiene practices

- keep steps explicit and deterministic
- pin major action versions
- fail fast on missing artifacts
- surface concise summary annotations for quick PR review

A clean pipeline is easier to evolve than a “works somehow” YAML maze.

## Examples/screenshots to include

- workflow graph showing artifact handoff
- successful vs failed run comparison
- Playwright trace triage screenshot
- runtime trend before/after optimization

## Actionable takeaway

Design your pipeline around artifact continuity. It usually gives the largest speed + reliability gain in one decision.
