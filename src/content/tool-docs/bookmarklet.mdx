<h5 class="mt-4">What Is a Bookmarklet?</h5>
<p>
  A bookmarklet is a browser bookmark whose URL begins with <code>javascript:</code> instead of
  <code>http://</code>. When clicked, the browser executes the embedded JavaScript on the current
  page — no browser extension or installation required. Common uses include toggling dark mode,
  extracting data from tables, tweaking page styles, or automating repetitive tasks.
</p>

<h5 class="mt-4">The Compilation Pipeline</h5>
<p>Your code passes through a strict, sequential pipeline:</p>
<ol>
  <li>
    <strong>IIFE Wrapper</strong> — The code is wrapped in
    <code>(function(){'{'}&nbsp;&hellip;&nbsp;{'}'})()</code> so local variables don't leak into the
    target page's global scope.
  </li>
  <li>
    <strong>Minification via Terser</strong> — The industry-standard
    <a href="https://terser.org" target="_blank" rel="noopener noreferrer">Terser</a>
    library handles comment stripping, whitespace removal, dead-code
    elimination, and constant folding. It uses a full JavaScript parser so edge
    cases like regex literals, template literals, and semicolon-free style are
    all handled correctly.
  </li>
  <li>
    <strong>URI Encoding</strong> — Characters that would break a URL bar
    (<code>%</code>, <code>"</code>, <code>'</code>, <code>#</code>, <code>&amp;</code>,
    <code>&lt;</code>, <code>&gt;</code>, <code>\`</code>) are percent-encoded so the browser
    doesn't mangle your script before it executes.
  </li>
</ol>

<h5 class="mt-4">Privacy</h5>
<p>
  All compilation runs entirely in your browser — your code never leaves your device.
</p>
