<h4>How It Works</h4>
<p>
  This tool runs a large language model <strong>entirely inside your browser tab</strong>
  using <a href="https://webassembly.org/" target="_blank" rel="noopener">WebAssembly</a>.
  No text you type is ever sent to a server — inference happens locally on your CPU.
</p>

<h5 class="mt-4">Powered by Transformers.js</h5>
<p>
  <a href="https://huggingface.co/docs/transformers.js" target="_blank" rel="noopener">Transformers.js</a>
  is the official JavaScript port of Hugging Face Transformers. It uses
  <a href="https://onnxruntime.ai/" target="_blank" rel="noopener">ONNX Runtime Web</a>
  to run pre-converted ONNX model files inside the browser's WebAssembly engine — no plugins or
  native installs required.
</p>

<h5 class="mt-4">Implementation Details</h5>
<ol>
  <li>
    On first use the browser downloads the selected model's ONNX weights from the
    <a href="https://huggingface.co/" target="_blank" rel="noopener">Hugging Face Hub</a>
    CDN and caches them in the browser's
    <a href="https://developer.mozilla.org/en-US/docs/Web/API/Cache" target="_blank" rel="noopener">Cache Storage</a>.
    Subsequent visits reuse the cached files with no re-download.
  </li>
  <li>
    Tokenization (text → token IDs) and de-tokenization (token IDs → text) run in the same
    WASM thread using the model's bundled vocabulary.
  </li>
  <li>
    The <a href="https://huggingface.co/docs/transformers.js/api/utils/generation#utilsgenerationtextstreamer" target="_blank" rel="noopener">TextStreamer</a>
    API streams generated tokens back to the page one at a time, so you see the response
    appear incrementally — just like server-side streaming APIs.
  </li>
  <li>
    For chat models (TinyLlama), the full conversation history is passed on every turn using
    the model's built-in chat template, keeping the assistant aware of prior context.
  </li>
</ol>

<h5 class="mt-4">Performance Expectations</h5>
<p>
  WASM inference is CPU-bound and noticeably slower than GPU-accelerated APIs.
  Typical generation speed on a modern laptop is <strong>1 – 10 tokens/second</strong>
  depending on the model and CPU. The first generation in a session may be slower
  as the WASM engine JIT-compiles the model graph.
</p>

<h5 class="mt-4">Privacy</h5>
<p>
  All inference runs locally in your browser tab. Your messages, model weights, and any
  generated text never leave your device.
</p>
