<h4>How It Works</h4>
<p>
  This tool compresses and converts images entirely inside your browser using
  <a href="https://imagemagick.org/" target="_blank" rel="noopener">ImageMagick</a> compiled to
  <a href="https://webassembly.org/" target="_blank" rel="noopener">WebAssembly</a> via the open-source
  <a href="https://github.com/dlemstra/magick-wasm" target="_blank" rel="noopener">magick-wasm</a> library.
  <strong>Your images are never uploaded to a server.</strong>
</p>

<h5 class="mt-4">Options</h5>
<div class="table-responsive">
  <table class="table table-sm table-bordered">
    <thead class="table-light">
      <tr><th>Option</th><th>Description</th></tr>
    </thead>
    <tbody>
      <tr>
        <td><strong>Output Format</strong></td>
        <td>
          Choose <strong>JPEG</strong> for photos (lossy, smallest files),
          <strong>PNG</strong> for graphics with transparency (lossless), or
          <strong>WebP</strong> for excellent compression with broad browser support.
        </td>
      </tr>
      <tr>
        <td><strong>Quality</strong></td>
        <td>
          Controls how much detail is retained versus file size reduction for lossy formats
          (JPEG and WebP). 80% is a good default; lower values produce smaller files with
          more visible artifacts. Quality has no effect for PNG, which is always lossless.
        </td>
      </tr>
      <tr>
        <td><strong>Limit max width</strong></td>
        <td>
          Scales the image down proportionally so that its width does not exceed the
          specified pixel value. The aspect ratio is preserved. Images already narrower
          than the limit are not scaled up.
        </td>
      </tr>
    </tbody>
  </table>
</div>

<h5 class="mt-4">Implementation Details</h5>
<p>
  The
  <a href="https://github.com/dlemstra/magick-wasm" target="_blank" rel="noopener">magick-wasm</a>
  library is loaded once from a CDN on the first compression. The ~14 MB WebAssembly binary
  is downloaded and compiled by the browser's built-in WebAssembly engine — no plugins or
  extensions required. Subsequent compressions within the same session reuse the already-loaded
  engine.
</p>
<p>
  Processing steps:
</p>
<ol>
  <li>The selected file is read into an in-memory <code>Uint8Array</code> via the
    <a href="https://developer.mozilla.org/en-US/docs/Web/API/FileReader" target="_blank" rel="noopener"><code>FileReader</code></a> API.</li>
  <li>ImageMagick decodes the image into an uncompressed pixel buffer.</li>
  <li>If a max width is set and the image is wider, it is resized proportionally.</li>
  <li>The quality setting is applied (JPEG / WebP only).</li>
  <li>ImageMagick re-encodes the image into the chosen format and returns a new
    <code>Uint8Array</code>.</li>
  <li>A preview is displayed using a
    <a href="https://developer.mozilla.org/en-US/docs/Web/API/URL/createObjectURL_static" target="_blank" rel="noopener">Blob URL</a>,
    and you can download the file.</li>
</ol>

<h5 class="mt-4">Privacy</h5>
<p>
  All image processing runs locally in your browser tab. No image data, file names, or
  metadata are ever sent to any server.
</p>
