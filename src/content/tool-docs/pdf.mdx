<h4>How It Works</h4>
<p>
  This tool adds or removes passwords from PDF files entirely inside your browser using
  <a href="https://github.com/qpdf/qpdf" target="_blank" rel="noopener">QPDF</a> compiled to
  <a href="https://webassembly.org/" target="_blank" rel="noopener">WebAssembly</a>.
  <strong>Your PDF is never sent to any server.</strong>
</p>

        </td>

<h5 class="mt-4">Implementation Details</h5>
<p>
  The QPDF WebAssembly binary (~1.3 MB) is fetched from a CDN on first use and instantiated
  inside a
  <a href="https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API" target="_blank" rel="noopener">Web Worker</a>
  so that the page stays responsive even for large files.
</p>
<ol>
  <li>The selected PDF is read into an <code>ArrayBuffer</code> in the browser.</li>
  <li>The bytes are posted to a Web Worker which writes them to the QPDF virtual filesystem
    (<code>FS.writeFile</code>).</li>
  <li>QPDF processes the file entirely in WebAssembly memory.</li>
  <li>The output bytes are read back (<code>FS.readFile</code>) and turned into a
    <code>Blob URL</code> for download — no disk writes, no network requests.</li>
</ol>

<h5 class="mt-4">Privacy</h5>
<p>
  All processing runs locally in your browser tab.
  Your PDF bytes, passwords, and any sensitive content are never transmitted anywhere.
</p>
