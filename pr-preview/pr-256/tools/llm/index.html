How It Works

This tool runs a large language model entirely inside your browser tab with WebGPU acceleration when available (and WebAssembly fallback when it is not). No text you type is ever sent to a server — inference happens locally on your device.

Powered by Transformers.js

Transformers.js is the official JavaScript port of Hugging Face Transformers. It uses ONNX Runtime Web to run pre-converted ONNX model files directly in the browser with GPU-first execution and no native installs required.

Implementation Details
  1. On first use the browser downloads the selected model’s ONNX weights from the Hugging Face Hub CDN and caches them in the browser’s Cache Storage. Subsequent visits reuse the cached files with no re-download.
  2. The runtime first attempts the WebGPU backend for faster generation and automatically falls back to WebAssembly if your browser/device does not support WebGPU.
  3. Tokenization (text → token IDs) and de-tokenization (token IDs → text) run locally using the model’s bundled vocabulary.
  4. The TextStreamer API streams generated tokens back to the page one at a time, so you see the response appear incrementally — just like server-side streaming APIs.
  5. For chat models (Gemma 4), the full conversation history is passed on every turn using the model’s built-in chat format, keeping the assistant aware of prior context.
Performance Expectations

WebGPU is typically much faster than CPU-only WASM execution. If fallback to WASM is required, performance is slower and depends heavily on your CPU.

Privacy

All inference runs locally in your browser tab. Your messages, model weights, and any generated text never leave your device.
